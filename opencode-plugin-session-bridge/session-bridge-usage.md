# /peek — 跨会话内容读取

在当前会话中查看其他会话的对话内容。内容放入输入框，可编辑后再提交。不消耗 token。

## 安装

将以下两个文件放入对应目录：

```
~/.config/opencode/commands/peek.md
~/.config/opencode/plugins/session-bridge.ts
```

重启 OpenCode 生效。

## 用法

### 快速开始

```
/peek
```

自动在输入框生成一条完整命令（指向最近活跃的会话），直接回车即可执行。

### 查看指定会话的最新 AI 回答

```
/peek auth重构
```

### 查看更多轮次

```
/peek auth重构 3
```

拿到最近 3 轮 AI 回答。

### 查看完整对话（用户提问 + AI 回答）

```
/peek auth重构 3 all
```

### 只看用户提问

```
/peek auth重构 3 question
```

### 列出可用会话

```
/peek 5
```

列出最近 5 个会话（上限 10），输出为可直接执行的 `/peek` 命令。

### 会话名含空格

```
/peek "前端 开发" 3 all
```

支持中英文双引号：`""` `""` `「」`

## 参数格式

```
/peek [会话名] [数量] [过滤模式]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 会话名 | 最近活跃的会话 | 支持模糊匹配标题、ID 前缀 |
| 数量 | 1 | 获取最近几轮（上限 20） |
| 过滤模式 | answer | `answer` 只看 AI 回答，`question` 只看用户提问，`all` 全部 |

## 输出格式

**单轮**：直接显示纯文本，无额外标记。

**多轮**：用 header 分隔，序号倒序（1 = 最新），内容从老到新：

```
[ 3/3  AI说：]
第一轮回答内容...

[ 2/3  用户说：]
追问内容...

[ 1/3  AI说：]
最新回答内容...
```

## 注意事项

- 内容放入输入框后可自由编辑、删除、追加指令，按回车提交给 AI
- 一"轮"= TUI 中看到的一次连续回答（多条底层消息自动合并）
- Markdown 格式已自动转为纯文本
- 纯代码块/工具调用等无文本内容会自动跳过
