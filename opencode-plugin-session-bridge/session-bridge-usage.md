# /peek — 跨会话内容读取

在当前会话中查看其他会话的对话内容。内容放入输入框，可编辑后再提交。不消耗 token。

## 安装

将以下两个文件放入对应目录：

```
~/.config/opencode/commands/peek.md
~/.config/opencode/plugins/session-bridge.ts
```

重启 OpenCode 生效。

## 用法

### 快速开始

```
/peek
```

自动在输入框生成一条完整命令（指向最近活跃的会话），直接回车即可执行。输出格式如：

```
/peek 跨会话模型交互可行性 1 answer
```

修改数字可获取更多轮次，修改 `answer` 为 `all` 或 `question` 可切换过滤模式。

### 查看指定会话的最新 AI 回答

```
/peek auth重构
```

### 查看更多轮次

```
/peek auth重构 3
```

拿到最近 3 轮 AI 回答。

### 查看完整对话（用户提问 + AI 回答）

```
/peek auth重构 3 all
```

### 只看用户提问

```
/peek auth重构 3 question
```

### 列出可用会话

```
/peek 5
```

列出最近 5 个会话（上限 10），输出为可直接执行的 `/peek` 命令，最新的在最下面。

### 会话名含空格

```
/peek "前端 开发" 3 all
```

支持中英文双引号：`""` `""` `「」`

## 参数格式

```
/peek [会话名] [数量] [过滤模式]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 会话名 | 最近活跃的会话 | 支持模糊匹配标题、ID 前缀；优先匹配最近活跃的同名会话 |
| 数量 | 1 | 获取最近几轮（上限 20） |
| 过滤模式 | answer | `answer` 只看 AI 回答，`question` 只看用户提问，`all` 全部 |

## 输出格式

**单轮**：直接显示纯文本，无额外标记。

**多轮**：用 header 分隔，序号倒序（1 = 最新），内容从老到新：

```
[ 3/3  AI说：]
第一轮回答内容...

[ 2/3  用户说：]
追问内容...

[ 1/3  AI说：]
最新回答内容...
```

**question 工具问答**：如果对话中包含 AI 的交互式提问（选择题/确认框），会以 Q/A 格式展示问题、选项和答案：

```
Q: 对于行数的更新策略——代码行数会持续变化。你希望我怎么处理？
  1. 删除整节（推荐）：从文档中完全移除...
  2. 标注 [未实现]：保留内容但加标注...
  3. 实现它：根据文档描述实现代码逻辑...
A: 实现它
```

尚未回答的问题显示 `A: (待回答)`。

## 注意事项

- 内容放入输入框后可自由编辑、删除、追加指令，按回车提交给 AI
- 一"轮"= TUI 中看到的一次连续回答（多条底层消息自动合并）
- Markdown 格式已自动转为纯文本（使用 Bun 内置 GFM 解析器）
- 纯代码块/工具调用等无文本内容会自动跳过，不占轮次名额
- API 默认只返回最近 100 条 session，太久不活跃的对话可能找不到。此时会提示：请先切到目标对话发送任意消息刷新排序后重试
